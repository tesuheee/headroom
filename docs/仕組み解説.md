# ai-usage-webview2-portable — 仕組み解説

> Claude と OpenAI Codex の使用量をリアルタイム表示する Windows デスクトップウィジェット。

ビジュアル図: [architecture.html](./architecture.html) をブラウザで開くと全体像・データフロー・更新スケジュール・描画の詳細をインタラクティブに確認できます。

---

## 一言で言うと

**「Edge（WebView2）を非表示ブラウザとして使い、使用量ページをスクレイピングして、Windows Forms の小さなウィンドウに GDI+ でリッチ描画する」** アプリです。

---

## 技術スタック

| 項目 | 内容 |
|------|------|
| 言語 | C# (.NET Framework 4.6.2+) |
| UI | Windows Forms + GDI+ カスタム描画 |
| ブラウザエンジン | Microsoft Edge WebView2 (v1.0.3537.50) |
| ビルド | PowerShell + csc.exe（Visual Studio 不要）|
| 設定 | JSON（Regex で手動パース）|
| 依存関係 | WebView2 NuGet パッケージのみ |

## システム構成（4クラス）

```
Program.cs
├── UsageForm      : メインウィンドウ（スクレイピング・描画・スケジュール）
├── SettingsForm   : 設定ダイアログ（フォント・色・間隔など）
├── ServiceState   : サービス別ランタイム状態（Claude / Codex）
└── UsageData      : 使用量データ + Used⇔Remaining 相互変換メソッド
```

## データ取得の仕組み

### 1. 非表示ブラウザを2つ起動

起動時に `InitializeWebViewsAsync()` が呼ばれ、画面に見えない WebView2 インスタンスを Claude 用・Codex 用に1つずつ作成します。  
ログイン Cookie は `%LOCALAPPDATA%\AiUsageWebView2\WebView2Profile` に保存されるため、**一度ログインすれば以降は自動認証**です。

2サービスの取得は `Task.WhenAll()` で並列実行されます。

### 2. 使用量ページにナビゲート

`ScrapeAsync()` が対象 URL に遷移します。キャッシュを避けるため URL にランダムパラメータを付加し、`NavigationCompleted` イベントで読み込み完了を検知します（最大30秒待機）。

対象 URL:
- Claude: `https://claude.ai/settings/usage`
- Codex: `https://chatgpt.com/codex/cloud/settings/analytics#usage`

### 3. コンテンツ安定化ポーリング（重要）

`NavigationCompleted` 後も React 等 SPA の動的レンダリングが続く可能性があるため、**1.2秒 × 最大10回**ポーリングして内容が安定するのを待ちます。

「100文字以上あり、かつ前回取得テキストと一致」した時点で完了と判定します。単純に `NavigationCompleted` を待つだけでは空振りするケースに対応した工夫です。

### 4. JavaScript で DOM テキストを取得

`document.body.innerText` を JavaScript 経由で実行し、**レンダリング後のテキスト**を取得します。生 HTML ではなくテキストを使うことで、動的コンテンツにも対応しています。

### 5. 行分割 → セクション特定 → 正規表現パース

テキストを `SplitLines()` で行配列に変換し、`FindIndex()` でセクション見出し行のインデックスを特定します。セクションの開始〜終了範囲内だけを対象に % とリセット時刻を抽出します。

**Claude の例:**
- セクション特定: "現在のセッション" / "週間制限" の行
- 使用 %: `(\d+)\s*%\s*使用済み`
- リセット時刻: "4時間25分後にリセット" / "5/24 9:49"

**Codex の例:**
- セクション特定: "5時間の使用制限" / "週あたりの使用制限" の行
- 残り %: `(\d+)\s*%\s*(残り|remaining)?`

**Used/Remaining の相互変換**: Claude は使用 % を、Codex は残り % を返す（サービスによって形式が違う）。`UsageData` の計算プロパティが両方向変換し、表示モードに合わせて統一的に描画できます。

## 更新スケジュール（3段階）

```
[通常: 15分] ──────────────────── [ファイナル: 1分] ── [Boost: 1分]
                                    ↑ リセットまで15分切ったら自動発動    ↑ ユーザーが手動ON
```

| モード | 間隔 | 発動条件 |
|--------|------|---------|
| 通常 | 15分（デフォルト） | 常時 |
| ファイナルウィンドウ | 1分（デフォルト） | 5h リセットまで15分以内（自動）|
| ブーストモード | 1分（デフォルト） | ユーザーが手動 ON（15分間）|

10秒ごとの `schedulerTimer` が「更新が必要か？」をチェックし、タイミングが来たらリクエストを投げます。

## UI・描画の仕組み

### レイアウト: 左右2カラム + サイドレール

```
┌──────────────────────────────────────────────────────────┐
│  [ Codex カード              ] [ Claude カード            ] │×│
│    5時間: 残り 72% ████████░░  3h 42m                     │📌│
│    週間:  残り 55% ██████░░░░  5/24 9:49                  │⚙│
└──────────────────────────────────────────────────────────┘
```

右端の縦並び「×・📌・⚙」がサイドレール（`DrawSideRail()`）です。各カードのコントロール（トグル・更新アイコン）はカード内右上に配置。

### カード状態（3種類）

GDI+ の `LinearGradientBrush` でカード背景を塗り、状態に応じて色が変わります：

| 状態 | 背景グラデ | ボーダー | 備考 |
|------|-----------|---------|------|
| Normal | `#1E1E22 → #161618` | `#303036` | 上端に薄い白ハイライト |
| Stale（古いデータ） | `#26220E → #1E1C10` | `#504818` | "古い" バッジ表示 |
| Exhausted（上限到達） | `#2A1616 → #201010` | `#50282A` | アクセントが赤に変化・"上限" バッジ |

### バーの描画レイヤー（DrawBar）

バーは4層で描画されます：

1. **トラック背景** — `#14141A` の暗い溝
2. **グロー** — 半透明アクセントカラーをオフセットして浮かせる
3. **グラデーションフィル** — 上端明るく下端を本来色にした `LinearGradientBrush`
4. **トップハイライト** — 上半分に白 18% を重ねて「艶」を表現

### スピナーアニメーション

更新中は Unicode の四分円文字 `◜◝◞◟` を 250ms ごとに切り替えて回転を表現。待機中は `↻`。

### 色のルール（デフォルト）

- 残量 > 30% → アクセントカラー（Claude: 青 / Codex: 緑）
- 残量 ≤ 30% → 黄（Warning）
- 残量 ≤ 15% → 赤（Critical）

※ 閾値はすべて設定可能。

---

## デフォルト設定値 (WidgetSettings)

| キー | デフォルト | 説明 |
|------|-----------|------|
| `normalIntervalMinutes` | **15** | 通常の更新間隔（分）|
| `boostDurationMinutes` | **15** | Boost の継続時間（分）|
| `boostIntervalMinutes` | 1 | Boost 中の更新間隔（分）|
| `finalRefreshWindowMinutes` | 15 | ファイナルウィンドウ開始タイミング（分前）|
| `finalRefreshIntervalMinutes` | 1 | ファイナルウィンドウ中の更新間隔（分）|
| `warningRemainingPercent` | **30** | 黄色バーに切り替わる残量 %|
| `criticalRemainingPercent` | **15** | 赤バーに切り替わる残量 %|
| `claudeShowUsed` | **false** | Claude の表示モード（false = 残量表示）|
| `codexShowUsed` | **false** | Codex の表示モード（false = 残量表示）|

## デバッグ

取得テキストは以下に保存されます:
- `~\.ai-usage-widget\claude-webview2.txt` — Claude から取得した生テキスト
- `~\.ai-usage-widget\codex-webview2.txt` — Codex から取得した生テキスト
- `~\.ai-usage-widget\*-error.txt` — エラー発生時の詳細

ページの HTML 構造が変わって解析できなくなった場合は、このファイルを確認します。

## ポータブル設計のポイント

レジストリを一切使わず、フォルダをまるごとコピーすれば別 PC でも動きます。  
ターゲット PC に必要なのは **Microsoft Edge WebView2 Runtime**（Windows 10以降はほぼプリインストール済み）のみです。
