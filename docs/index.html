Headroom Architecture

Headroom は Claude Code と Codex の使用量を OAuth トークンで直接 API から取得し、 WinForms のカスタム描画で小さな常駐ウィジェットとして表示する Windows デスクトップアプリです。 CLI が入っている PC では CLI が保存したトークンを読み、入っていない PC では Headroom 自身が PKCE OAuth フローで認証してトークンを保存します。 リフレッシュトークンによるアクセストークンの自動更新も自前で行います。

Program Shape

実装は単一の Program.cs にまとまっています。クラスの責務は明確に分かれています。

Program / HeadroomOptions 起動時引数と HEADROOM_FIXTURE_DIR を読み、通常モードかフィクスチャモードかを決めます。
UsageForm メインウィジェット。スケジューリング、API取得、状態更新、描画、クリック操作を担当します。
SettingsForm 設定ダイアログ。ライブプレビュー、保存、キャンセル復元、設定/ログ場所表示を担当します。
ServiceState Claude / Codex それぞれの実行中状態、最終更新、Boost、バーアニメーション値を持ちます。
UsageData 5時間枠、週次枠、リセット時刻、ステータスをサービス非依存の形に正規化します。
WidgetSettings 表示モード、言語、サイズ、閾値、更新間隔を settings.json に保存します。

認証情報の書き込み: CLI（claude / codex）がインストールされている場合は、認証情報の書き込みは CLI に任せ、Headroom は読み取り専用です。CLI が無い場合は Headroom 自身が PKCE OAuth フローで取得し、CLI 互換形式で .credentials.json / auth.json に書き込みます。
トークン更新: アクセストークン期限切れ時または 401 応答時に refresh_token で自動更新し、ファイルを上書きします。再ログインはほぼ発生しません。

Login Flow

ログインボタンの動作は CLI 検出結果で分岐します。

CLI あり cmd.exe /k で claude または codex login を起動。タイトルとプリバナーで「認証後はこのウィンドウを閉じてOK」を表示。
CLI 無し（PKCE） PKCE verifier/challenge を生成 → HttpListener を localhost:動的ポート で開き、システム既定ブラウザで認可URLを開く → /callback 受信 → トークン交換 POST → 認証ファイルに保存。

Fixture Mode

実際の利用枠を消費せず、UI状態をローカル JSON で再現します。JSON を編集すると FileSystemWatcher 経由でアプリが自動再描画し、スクリーンショットで確認できます。

Debug app
.\build.ps1 -DebugFixture
→
debug\Headroom.fixture.exe
→
--fixture <フォルダ>
→
Screenshot verification

指定フォルダには本番 API レスポンスと同じ形の claude.json / codex.json を置きます。 ステータス検証用には "status": "login_required" を含む JSON を置けば login_required 状態を再現できます。

Build

Production build .\build.ps1
bin\Headroom.exe を生成します。
Fixture build .\build.ps1 -DebugFixture
debug\Headroom.fixture.exe を生成します。

通常 exe が起動中で bin\Headroom.exe をロックしている場合でも、 debug 出力なら検証用 exe を作れます。debug/ は配布物にも Git にも含めません。

Maintenance

API変更に強くする: パーサーは現在 Regex ベースです。APIレスポンスの形が変わった場合は ParseClaudeApi / ParseCodexApi と fixture を同時に更新します。
UI回帰を潰す: 新しい表示状態を見つけたら、まず docs/fixtures/ にシナリオを追加し、 debug\Headroom.fixture.exe でスクリーンショット確認します。
