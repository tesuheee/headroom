データ取得〜表示までのフロー
1
🚀 起動・WebView2 初期化
InitializeWebViewsAsync() が非表示 WebView2 を2インスタンス作成。
プロファイルは %LOCALAPPDATA%\AiUsageWebView2\WebView2Profile に保存され、 Cookie が持続するので一度ログインすれば次回以降は自動認証。
2サービスは Task.WhenAll() で並列取得。
2
🌐 ページナビゲーション
ScrapeAsync() が URL にキャッシュバスター付きで遷移。
NavigationCompleted イベント待ち（最大 30 秒タイムアウト）。
3
⏳ コンテンツ安定化ポーリング （重要）
NavigationCompleted 後も動的レンダリングが続く可能性があるため、 1.2秒 × 最大10回 ポーリングして内容が安定するのを待つ。
「100文字以上あり、かつ前回と同じテキスト」になった時点で完了と判定。
→ SPA（React 等）で遅延描画されるコンテンツにも対応する工夫。
4
📄 JavaScript で DOM テキスト取得
document.body.innerText を JS 実行で取得 → DecodeJsonString() でエスケープ解除。
生 HTML ではなくレンダリング済みテキストなので、CSS で非表示の要素はスキップされる。
5
🔍 行分割 → セクション特定 → 正規表現パース
SplitLines() でテキストを行配列に分割し、FindIndex() でセクション見出し行を特定。
セクションの開始〜終了インデックスの範囲内だけを対象に % と リセット時刻を抽出する。
Claude: "現在のセッション" / "週間制限" → (\d+)\s*%\s*使用済み
Codex: "5時間の使用制限" / "週あたりの使用制限" → (\d+)\s*%\s*(残り|remaining)?
6
📊 UI 更新 (GDI+ 再描画)
ServiceState / UsageData を更新 → Invalidate() を呼び出す。
250ms タイマーが常時 OnPaint() → DrawService() を駆動（スピナーアニメも同じタイマー）。
7
⚠️ エラー・デバッグ
ログイン未完了 → "ログイン" バッジ表示 → クリックでログインダイアログ。
取得テキストは ~\.ai-usage-widget\{name}-webview2.txt に毎回保存。
例外は {name}-error.txt に記録。パース変化があってもファイルで追跡できる。

🟢 通常モード (Normal)
15 分
デフォルト: normalIntervalMinutes = 15
10秒ごとの schedulerTimer が「経過時間 ≥ 設定間隔？」をチェックし、条件を満たしたらリクエスト投入。
🟡 ファイナルウィンドウ (Final)
1 分
デフォルト: finalRefreshIntervalMinutes = 1
5hリセットまで残り 15分（finalRefreshWindowMinutes）を切ると自動発動。制限解放の瞬間を素早くキャッチ。
🔴 ブーストモード (Boost)
1 分
デフォルト継続時間: boostDurationMinutes = 15
ユーザーがトグルで手動 ON。15分間（設定可）だけ1分更新。OFF にするか時間切れで自動終了。
🎨 描画タイマー (Paint)
250 ms
固定値（変更不可）
UI 専用タイマー。spinnerFrame を 0→1→2→3→0 と更新して回転アニメ。データ取得とは完全独立。
判定ロジック (RefreshIntervalMinutes)
if (service.BoostUntil != null) {
    // Boost ON → 短い間隔
    return BoostIntervalMinutes;  // 1分
} else if (resetRemaining < FinalWindow) {
    // 5h リセットが近い → ファイナルウィンドウ
    return FinalRefreshIntervalMinutes; // 1分
} else {
    // 通常時
    return NormalIntervalMinutes;   // 15分
}

設計上のこだわり
📦 ポータブル設計
レジストリ不使用。bin/ フォルダごとコピー可。設定は settings.json、Cookie は WebView2 プロファイルに保存。
🚫 外部ライブラリ最小化
JSON は Regex で手動パース。GDI+ で完全カスタム描画（WPF 等不使用）。依存は WebView2 NuGet のみ。
⏳ SPA 対応の安定化ポーリング
NavigationCompleted 後も 1.2 秒 × 最大 10 回ポーリングして「テキストが落ち着いた」ことを確認してからパース。React 等の遅延レンダリングに対応。
🔀 Used/Remaining の相互変換
Claude は使用% を返し、Codex は残量% を返す（サービスによって違う）。UsageData の計算プロパティが両方向変換し、表示モード（showUsed）に合わせて統一的に描画できる。
